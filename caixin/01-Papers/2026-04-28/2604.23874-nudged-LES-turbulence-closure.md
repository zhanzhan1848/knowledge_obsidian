## 核心创新点

### 解决的核心问题
传统 a-posteriori 深度学习湍流闭合的三大痛点：
1. **高计算成本**：需通过 LES 求解器反向传播梯度
2. **求解器修改**：需要对已有求解器大幅改造
3. **泛化性差**：跨不同数值格式表现不佳

### 新方法：Nudged LES (Continuous Data Assimilation)
- **核心思想**：将 DNS 数据作为稀疏观测，用 nudging 强制 LES 接近真实解
- **训练方式**：a-priori 训练 + 无需伴随/梯度反传
- **稳定性**：长期稳定，不依赖数值格式

### 数学框架
$$\frac{\partial \bar{u}}{\partial t} + \bar{u}\cdot\nabla\bar{u} = -\frac{1}{\rho}\nabla\bar{p} + \nu\nabla^2\bar{u} - \underbrace{\mathcal{N}(x,t)}_{\text{neural network forcing}}$$

Nudging 项: $\mathcal{N} = \lambda W (u_{DNS} - \bar{u})$

## 控制方程

### 过滤 N-S (LES)
$$\frac{\partial \bar{u}_i}{\partial t} + \bar{u}_j\frac{\partial \bar{u}_i}{\partial x_j} = -\frac{1}{\rho}\frac{\partial \bar{p}}{\partial x_i} + \nu\frac{\partial^2 \bar{u}_i}{\partial x_j^2} - \frac{\partial \tau_{ij}}{\partial x_j} + \mathcal{N}_i$$

### 子网格应力
$$\tau_{ij} = \overline{u_i u_j} - \bar{u}_i\bar{u}_j$$

## 实验设置

- **DNS 数据**: 强制各向同性湍流
- **LES 求解器**: 多种格式测试
- **网络架构**: 全连接/卷积网络
- **评估指标**: 能谱、统计量、长时间稳定性

---

## 关键发现

1. ✅ 成功学习匹配真实统计所需的 forcing
2. ✅ 跨数值格式泛化能力
3. ✅ 无需 adjoints 或梯度反传
4. ⚠️ 高阶格式依赖仍需进一步研究

---

## 对 CFD 的意义

1. **实用性强**：可集成到现有 LES 求解器
2. **端到端可微**：更易部署
3. **不确定性传递**：为 uncertainty-aware 湍流建模提供路径
