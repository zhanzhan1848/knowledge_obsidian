## 核心贡献

> 首个将文献检索、代码编译、视觉物理验证和论文写作整合在一个可检查工作流中的 CFD AI Agent 系统

1. **三通道耦合架构**: 参数扫描 (固定求解器) + 本地 C++ 编译 (新物理模型) + 开放假设搜索 (对照比较器)
2. **视觉-语言物理验证门 (Vision-Language Physics-Verification Gate)**: 渲染流场图像后检查，检测 solver 日志无法发现的场级别故障
3. **Spalart-Allmaras 运行时校正自动发现**: 周期性 hills Re_h=5600 降低壁面 Cf RMSE 7.89%

---

## 技术方案

### 核心思想

使用 LLM-based agent 扩展科学发现闭环到高保真物理模拟器。关键挑战：
- 求解器完成 ≠ 物理有效性
- 许多故障仅出现在场级图像而非 solver 日志

### 关键技术

| 技术 | 说明 |
|------|------|
| Foam-Agent | OpenFOAM 自动化接口 |
| 视觉验证门 | VLM 检查渲染流场 |
| LLM 主干 | GPT-5.5 |
| 三通道框架 | 参数扫描 / C++ 编译 / 假设搜索 |
| 植物故障 ablation | 验证视觉门检测 14/16 静默故障 |

### 系统架构

```
文献检索 → 生成假设 → Foam-Agent 执行
                              ↓
                    CFD solver (OpenFOAM)
                              ↓
                    渲染流场 → VLM 验证 ← 失败? → 重运行
                              ↓
                         通过 → 写论文
```

## 实验结论

- **任务**: 5 个跨不同配置的 CFD 任务
- **基线**: ARIS, DeepScientist
- **结果**: 
  - Spalart-Allmaras 校正 → 降低 wall Cf RMSE 7.89% (vs DNS)
  - 视觉门检测 14/16 静默故障
  - 在匹配 LLM 成本下，两基线执行部分 CFD 工作流但缺乏领域有效性门控

---

## 局限性

- 依赖 OpenFOAM (不可直接移植到 SU2/Basilisk)
- 视觉验证门需要高质量渲染，对低分辨率模拟效果有限
- 尚未验证端到端完整论文生成
