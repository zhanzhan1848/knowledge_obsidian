## 核心贡献
1. **GPU 数据并行软件简化框架**：RaFI 简化了需要光线或类似工作项在 GPU 间迁移的多 GPU/多节点 GPU 软件构建
2. **简单 CUDA+MPI 接口**：提供简单接口让 CUDA 内核将工作项转发到其他 GPU，自动管理所有 CUDA 和 MPI 相关工作
3. **通用射线追踪基础设施**：展示了在多个示例应用中的潜力

## 技术方案
### 核心接口
```cpp
// RaFI 转发接口
rafi_forward_workitem(work_item, dest_gpu_id);
// RaFI 自动处理：
// - GPU 间内存拷贝
// - MPI 通信
// - CUDA 流同步
```

### 实现
- CUDA kernel 发起转发请求
- RaFI runtime 管理跨 GPU/MPI 的工作调度
- 目标 GPU kernel 接收并处理工作项

## 实验结论
- 有效减少多 GPU 射线追踪应用的开发复杂度
- 自动管理通信和同步，程序员无需关心细节
- 适用于多种射线追踪应用场景

## 局限性
- 依赖 NVIDIA GPU + MPI 环境
- 额外通信开销
- 需要工作项可序列化

## 可行性分析
- 实现难度：中（基础设施，适合集成而非从头实现）
- 性能预期：取决于应用和网络
- 适用场景：大规模射线追踪、多 GPU 渲染集群

## 相关工作
- Multi-GPU rendering
- GPU cluster computing
- CUDA IPC
- MPI-based parallelism

## 笔记
