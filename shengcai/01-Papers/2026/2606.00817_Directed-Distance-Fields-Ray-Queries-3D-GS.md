## 核心贡献
1. **将 3DGS 场景转为射线 oracle**：通过蒸馏有向距离场（DDF）将训练好的 3DGS 场景转化为射线查询预言机
2. **常数时间/空间复杂度**：DDF 是小神经场（52MB），查询时间和内存不随场景 Gaussians 数量增长
3. **常时查询优势**：DDF 比球追踪等效 SDF 快 26~72 倍；与 BVH 相比，即使在专用 RT-core 硬件上，查询时间和内存也不随场景增长
4. **支持全局光照 secondary rays**：作为 secondary-ray oracle，实现参考级射线追踪阴影（30.3dB）和环境光遮蔽（21.3dB）

## 技术方案
### DDF 定义
```
DDF(ray_origin, ray_direction) → (distance_to_surface, hit_flag)
```
每个查询 = 一次前向传播，52MB，查询时间和内存不依赖 Gaussian 数量。

### 监督信号研究
- **深度监督不足**：从 Gaussians 渲染的深度太模糊，无法教到薄结构
- **干净距离监督有效**：SDF 监督能恢复薄结构

### 完整管线（无需网格）
```
Images → 3DGS scene → Neural surface → Clean distances → DDF learning
```

## 公式
DDF 查询：
```math
(d, h) = DDF_{\phi}(\mathbf{o}, \omega)
```

其中 $d$ 是到最近表面的距离，$h \in \{0,1\}$ 表示是否命中。

全局光照应用：
```math
L_{GI} = \int_{\Omega} V(\mathbf{x}, \omega_i) L_i(\mathbf{x}, \omega_i) (\omega_i \cdot \mathbf{n}) d\omega_i
```

其中可见性 $V$ 由 DDF 查询得到。

## 实验结论
| 指标 | 数值 |
|------|------|
| 阴影质量 (PSNR) | 30.3 dB |
| AO 质量 (PSNR) | 21.3 dB |
| 速度提升（vs 球追踪 SDF） | 26~72× |
| DDF 模型大小 | 52 MB |
| 测试数据集 | 142 objects |

## 局限性
- 需要干净的 SDF 监督（从 neural surface 提取）
- 对动态场景需要重新训练 DDF
- 52MB 固定内存开销

## 可行性分析
- 实现难度：中（需要 3DGS + SDF + 神经场训练）
- 性能预期：常数时间查询，支持实时 secondary rays
- 适用场景：3DGS 场景的全局光照、阴影、环境光遮蔽
