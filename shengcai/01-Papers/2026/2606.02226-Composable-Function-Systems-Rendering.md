## 核心贡献
1. 提出使用 function systems 进行通用（非分形）可视化和模拟
2. 引入 Quibble：GPU 上组合 function systems 的元编程框架
3. 核心优势：运行时性能、拓扑非平凡对象创建、与其他图形算法的互操作性

## 技术方案

### 核心思想
Function systems 是创建和操作复杂对象自然语言，可在 GPU 或 CPU 上保持最小内存。本文将分形技术扩展到通用可视化和模拟。

### 应用场景
- 低帧率动画中间帧控制
- 点云可控变形
- 复杂动画工作流的元编程

### 代码框架
```cpp
// Quibble 框架概念
auto system = compose(
    transform(translate(...)),
    sdf(shape1),
    sdf(shape2)
);
render(system);
```

## 可行性分析
- 实现难度：高（需要元编程和 GPU 编程经验）
- 适用场景：高级视觉效果、动画控制
