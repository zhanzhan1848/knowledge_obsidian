# Monte Carlo Steklov Operators for Large-Scale Geometry Processing in the Wild

## 论文信息
- **arXiv**: [2606.05581](https://arxiv.org/abs/2606.05581)
- **作者**: (from Objaverse dataset study)
- **日期**: 2026-06-03
- **会议**: -
- **主题**: Steklov operator, Dirichlet-to-Neumann, Monte Carlo, spectral analysis, mesh learning

## 问题背景

### 内在方法局限
内在方法（如 Laplacian）是几何处理工具箱的默认选项，特别适合需要等距不变性的算法（形状分析、学习、编辑）。

但内在方法在**野外几何（in-the-wild geometry）**场景下假设会变得脆弱：
1. 网格质量无法保证
2. 多连通组件建模

### 体积构建优势
在这种情况下，**体积构建（volumetric constructions）**定义更好，因为可以放宽表面拓扑限制。

## 核心方法

### Dirichlet-to-Neumann (DtN) 算子
提出估计 **DtN 算子**（边界到边界的体积算子）及其相关 **Steklov 特征模**的蒙特卡洛方法。

### 关键创新
1. 将边界算子本身作为估计目标
2. DtN 算子通过体积随机过程定义
3. 推广到外部域，耦合通过周围环境空间的断开组件

### 速度提升
比现有**边界元方法（BEM）**计算 Steklov 谱快**数个数量级**，同时对低质量三角化、高分辨率网格和多组件几何保持鲁棒。

## 大规模验证
- 计算约 **450,000** 形状的内外 Steklov 特征谱（来自 Objaverse 数据集）
- 提出 **Steklov-CLIP**：基于网格的神经网络，使用体积谱算子进行大规模对比 3D 表示学习

## 算法流程
```
点云/网格 → Monte Carlo 采样 → DtN 算子估计 → Steklov 特征模 → 3D 表示学习
```

## 复杂度分析
- 时间复杂度：O(n) Monte Carlo sampling
- 空间复杂度：O(k) for k spectral components
