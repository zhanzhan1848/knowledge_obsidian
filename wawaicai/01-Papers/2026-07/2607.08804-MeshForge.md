## 核心贡献

> 提出神经符号程序合成方法，通过少量示例演示合成可复用的碰撞网格批量编辑程序

1. **编程-by-示例 (PBE)**: 用户演示编辑意图，合成可泛化的程序
2. **神经符号方法**: 多模态模型感知 + 符号组件泛化编辑逻辑
3. **高合成成功率**: 24 任务中 23/24 成功，平均 2.2 次演示 + 3.5 秒
4. **处理异构性**: 跨几何异构网格的批量编辑

---

## 技术方案

### 核心思想

碰撞网格定义物理引擎用于碰撞检测的几何，通常通过凸分解从视觉网格自动生成。但几何近似常不足以捕捉预期交互行为，需要人工后编辑。

**挑战**:
- **C1 异构性**: 同类资产碰撞网格在建模风格、分解粒度上差异大
- **C2 语义标签不足**: 标签太粗糙，需结合结构和空间信息
- **C3 感知噪声**: 多模态模型偶尔错误标注导致程序合成失败

**程序模型**: 一个或多个 extractor-action 对
- **Extractor**: 识别目标 hull 的逻辑 (语义标签 + 结构/空间条件)
- **Action**: 应用到 hull 的编辑操作 (删除、缩放、移动等)

**合成流程**:
```
输入: 同类资产的碰撞网格集合 + 用户演示

1. 感知阶段:
   - 多模态模型为每个 hull 预测语义标签
   - 记录用户对哪些 hull 进行了哪些操作

2. 符号合成阶段:
   - 基于标签 + 演示生成 extractor 候选
   - 过滤感知噪声 (soft matching)
   - 选择覆盖所有演示的最简 extractor

3. 验证:
   - 在所有演示网格上验证 extractor 正确性
   - 通过则输出程序
```

### 关键技术

| 技术 | 说明 |
|------|------|
| 多模态感知 | 预训练多模态模型识别 hull 语义标签 |
| 符号 extractor | 泛化编辑逻辑，过滤感知噪声 |
| Soft Matching | 处理感知噪声，对噪声标注的 hull 软化匹配 |
| 结构+空间条件 | 补充语义标签的细粒度条件 |
| PBE 编程范式 | 少量演示驱动程序合成 |

## 实验结论

- **数据集**: 24 个任务，600 个碰撞网格
- **指标**: 合成成功率、演示次数、合成时间
- **结果**: 23/24 任务成功，平均 2.2 演示 + 3.5 秒合成时间
- **应用**: 抽屉、杯子等日常物体的碰撞网格批量编辑

---

## 局限性

- 依赖多模态模型的感知质量
- 当前聚焦于凸 hull 编辑
- 复杂语义编辑可能需要更多演示
