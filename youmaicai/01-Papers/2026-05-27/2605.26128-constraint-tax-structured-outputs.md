## 核心贡献
1. **Constraint Tax**: 测量结构化输出约束对答案准确率和可执行准确率的影响
2. 揭示硬输出约束对小模型的假设不成立
3. 提出 "reason free, constrain late" 设计模式
4. 15,000 次 commodity-GPU 生成实验数据

## 关键发现
- **Schema validity**: 61.5% → 100.0% (hard schema decoding)
- **Answer accuracy**: 19.7% → 11.0% (下降!)
- **Wrong-valid-schema outputs**: 49.5% → 88.9% (大幅增加!)
- Qwen2.5-1.5B 工具调用: prompt-only JSON 91.5% → hard schema 48.0%

## 核心论点
- 硬输出约束对小模型不安全
- 错误是语义性的，非结构性的
- 3B 边界仍需支付 direct-schema tax
- 延迟打包 (delayed packaging): reason free, constrain late

## 实践建议
生产系统应分开报告:
1. Schema validity
2. Answer accuracy
3. Executable accuracy
4. Wrong-valid-schema rate
