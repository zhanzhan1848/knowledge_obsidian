## 核心贡献
1. **GEM (Governed Evolving Memory)**: 将长期 agent memory 视为新的数据管理工作负载
2. **四个状态级操作符**: ingestion, revision, forgetting, retrieval
3. **六个正确性条件**: 约束状态如何演化
4. **MemState 原型**: 基于 property-graph backend 实现
5. 揭示 no record-level system 可满足这些条件

## 四个失败模式
1. Unregulated growth
2. Missing semantic revision
3. Capacity-driven forgetting
4. Read-only retrieval

## 核心观点
- 正确性是状态轨迹 (state trajectory) 的属性，而非个别记录
- 现有数据库范式 (record-level) 无法满足长期记忆需求
- 需要状态级 (state-level) 操作符

## 研究方向
三个定义 memory-centric 数据管理作为工作负载的研究方向
