## 核心贡献
1. **因果推断视角**: 分析优化后 prompt 在不同任务间的泛化差异
2. **编辑模式识别**: 发现 consistent task-conditioned edit patterns
3. **关键发现**: complexity-increasing 和 meta-instructional edits 与数学/多跳推理负相关
4. **正向发现**: step-by-step 和 meta-cognitive edits 改善逻辑/序列推理任务
5. 揭示 prompt 优化失败源于 edit families 与 task characteristics 的系统性交互

## 关键洞察
- 优化 prompt 在一个 benchmark 的优势常无法迁移到另一个
- 即使切换 LLM backbone 也有此限制
- 非随机优化artifact，而是系统性交互

## 方法
- propensity-adjusted associational analysis
- 多种 prompt edits 的互补表示
- 跨优化框架、LLM backbone、NLP benchmark 综合分析
- cognitive-load annotations, surface-level text features, edit-motif analyses

## 实践建议
- task-conditioned optimizer design
- 考虑 edit families 与 task characteristics 的匹配
