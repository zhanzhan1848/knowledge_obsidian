# 🥬 LLM 论文分析: Psy-CoT + RAPO — Psychology-Grounded Role-Playing Agents

## 基本信息
- **arXiv**: [2606.27025](https://arxiv.org/abs/2606.27025)
- **类别**: cs.CL
- **日期**: 2026-06-25

## 核心贡献
1. **Psy-CoT**: Psychology-grounded Chain-of-Thought 框架，三步角色推理
2. **RAPO**: Role-Aware Policy Optimization，防止 reward model hacking
3. **超越现有 Role-Playing CoT 方法**: 在 CoSER, CharacterBench, CharacterEval 上验证

## 问题背景
### SFT 在 Role-Playing Agent 上的局限
- 鼓励行为模仿（behavioral mimicry）
- 缺乏深层、类人的内部思维过程
- **OOD 泛化能力差**

### LLM-based Reward Model 的 Hacking 问题
- Generic phrases（通用短语）和 genuinely role-specific phrases 收到相同梯度信号
- Hacking 随训练累积 → 模型无法区分两者

## Psy-CoT: 心理学链式推理

### 三步推理:
1. **Interaction Perception** (交互感知)
   - 理解当前对话情境和对方特征

2. **Psychological Empathy** (心理共情)
   - 从角色设定出发进行心理建模

3. **Logical Construction** (逻辑构建)
   - 生成符合角色的响应

### 核心思想
> 模型应从角色设定中**动态思考**，而非仅模仿表面模式

## RAPO: Role-Aware Policy Optimization

### 问题形式化
```
Generic phrase: P(phrase|role) ≈ P(phrase|any_role)
Role-specific phrase: P(phrase|role) >> P(phrase|any_role)
```

### 解法: Profile-Token Mutual Information Weighting
- **Positive advantage 时**: 放大 role-specific tokens 的梯度
- **Negative advantage 时**: 衰减 generic tokens 的梯度

### 数学表达
```math
Gradient_weight(token) = MI(profile; token) / baseline_gradient
```

## 实验结果

### 基准测试
- CoSER
- CharacterBench
- CharacterEval

### 主要发现
- Psy-CoT 优于现有所有 Role-Playing CoT 方法
- RAPO 在多个模型规模上一致地超越 GRPO
- GRPO 在训练中会逐渐向 generic phrases 漂移，RAPO 有效抑制了这个问题

## 核心洞察
> Reward model hacking 不仅是一个 RL 问题，也是一个**表示学习**问题

## 局限性
- 依赖高质量的角色设定描述（profile quality）
- 在极简单角色（几句话描述）上效果可能有限
