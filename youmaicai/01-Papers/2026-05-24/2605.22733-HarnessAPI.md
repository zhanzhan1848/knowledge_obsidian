## 核心贡献
1. 统一流式 HTTP endpoint（SSE）、OpenAPI/Swagger UI 和 MCP tool 从单一 handler + Pydantic schemas
2. 消除 LLM tool 部署的重复（HTTP endpoint + MCP tool registration）
3. 减少 74% 框架 boilerplate

## 问题背景
- Python function 作为 LLM tool 需要两种形式：HTTP endpoint + MCP tool registration
- 两者共享业务逻辑但周围 machinery 不同（routing, validation, serialisation, streaming, schema）
- 随代码演化容易 drift

## 方法
- 以 typed skill folder 为单一真相来源
- 双模内容协商：同一 handler 服务 SSE-streaming 和 JSON-returning clients
- 动态代码生成确保 Pydantic 类型注解传播至 FastMCP

## 核心创新点
- 基于 skill 的单一来源设计
- 零配置 MCP tool
- 继承 FastAPI 全部 middleware 和部署生态
