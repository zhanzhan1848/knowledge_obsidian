## 核心贡献
1. **Distribution Vector 概念**：将每个数据集的持续预训练效果表示为参数偏移向量 τᵢ = θ_CPTᵢ - θ_pt，而非训练一个混合数据模型
2. **OptiMer 框架**：用贝叶斯优化（TPE）自动搜索最优合并权重，将数据配比选择从训练前移到训练后
3. **向量池复用**：同一组 distribution vectors 可针对不同目标重新优化，无需重新 CPT

## 模型架构

### Distribution Vector 定义
```
τᵢ = θ_CPTᵢ - θ_pt
```
从预训练基模型 θ_pt 开始，在每个数据集 Dᵢ 上独立 CPT，得到 θ_CPTᵢ，提取参数偏移向量。

### 多向量合并公式
```
θ_merge = θ_pt + α_it · τ_it + Σᵢ αᵢ · τᵢ
```
τ_it 是 instruction-tuned 向量，合并后恢复指令遵循能力。

## 关键发现
- **向量近似正交**：不同分布的 distribution vectors cosine similarity 仅 0.03–0.31，支持线性组合
- **搜索效率提升 15–35×**：TPE 只需 ~100 次trial，而 DataMix 单次运行需 128.9 GPU 小时
- **负权重有效**：优化出的权重可以为负，用于消除跨分布干扰
- **权重可解释为数据配比**：用最优权重反推数据混合比例，重新训练 DataMix 也能提升性能

## 实验结果
| 任务组 | DataMix | OptiMer |
|--------|---------|---------|
| Ja+Math Avg | 67.86 | **69.98** (+2.1) |
| Ja+Code Avg | 67.23 | **69.68** (+2.5) |
| Ja+Zh+Math Avg | 63.71 | **70.37** (+6.7) |

基准：Gemma 3 27B，覆盖 16 个 benchmarks（English, Japanese, Chinese, Math, Code）

## 方法优势
1. **解耦训练与配比搜索**：每个分布独立训练，互不干扰
2. **后验优化**：训练完成后才优化合并权重，不浪费 GPU 时间
3. **目标定制**：同一向量池可产出针对不同任务优化的多个模型
4. **冷启动少**：仅需 N₀=20 随机trial 初始化 TPE

## 局限性
- 需要每个分布独立训练 N 个 CPT 模型（总计算量与 DataMix 相同）
- 当前仅在 Gemma 3 27B 上验证
