## 核心贡献

1. **问题**：单一 ReAct Agent 受限于单条长轨迹和有限上下文，难以同时处理深度和广度搜索
2. **方案**：提出 **WebSwarm**，一种渐进式递归委托框架，动态实例化 Agent 搜索节点，每个节点耦合局部目标和搜索模式
3. **创新点**：引入 Web 结构探测（probing）来引导节点扩展方向，以及同层兄弟节点间的过程经验复用

## 模型架构

### 核心机制：递归委托
```
Root Agent → 接收原始任务 → 创建搜索节点
节点接收 (local_objective, search_mode)
- 节点可自行解决
- 或递归委托子节点，组织本地协作结构（并行分治、顺序搜索验证、多路径采样聚合）
- 完成后返回证据和结果给父节点，父节点决定进一步扩展、修正或终止
```

### 搜索模式类型
- **Fact Lookup**: 快速可靠执行
- **Wide Search**: 多项覆盖
- **Deep Search**: 迭代线索发现与验证
- **Open-set Enumeration**: 在未知集合边界下平衡召回率和精确率

### 两类引导信号
1. **外部 Web 信息结构**：轻量级探测 Web 信息组织方式（集中 vs 分散），指导扩展方向
2. **内部过程经验**：同一父节点下同构搜索节点间的轨迹经验蒸馏

### 与现有方法对比
| 方法 | 递归深度 | 协作适应性 | 证据驱动扩展 |
|------|------|------|------|
| 现有 Multi-Agent | 仅根层分解 | 单一全局协作范式 | 按查询表面语义分解 |
| WebSwarm | 渐进递归扩展 | 节点级异构协作模式 | 按 Web 信息结构驱动 |

## 实验结果

### Benchmark: BrowseComp-Plus, WideSearch, DeepWideSearch, GISA
- 任务类型: Deep Search、Wide Search、Hybrid Deep-Wide Search
- WebSwarm 在所有四类基准上持续优于 ReAct Agent 和 Multi-Agent 基线

### 关键发现
- 深度-广度混合任务上优势最显著
- Web 结构探测显著减少冗余检索
- 经验复用对同构子节点有正向迁移效果

## 局限性
- 递归深度增加带来额外延迟开销
- 对异构节点的经验复用效果有限
