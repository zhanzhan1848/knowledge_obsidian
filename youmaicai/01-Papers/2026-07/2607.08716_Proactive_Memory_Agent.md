## 核心贡献

1. **问题识别**：提出 **Behavioral State Decay（行为状态衰减）** 作为长程 Agent 的核心失败模式——决策相关信息随轨迹增长逐渐失去对行为的影响
2. **架构设计**：提出 **双 Agent 记忆干预架构**，一个独立的 Memory Agent 伴随 Action Agent 运行，维护结构化记忆库并决定是否注入记忆提醒
3. **可学习策略**：使用 SFT + GRPO 在 Qwen3.5-27B 上训练开放权重的记忆策略，初步证明干预策略可学习

## 模型架构

### 问题设定
- Action Agent 策略: $a_t \sim \pi_A(a_t \mid x, \tau_{<t})$，其中 $\tau = (o_1, a_1, o_2, a_2, ..., o_T)$
- Memory Agent 在每 N 步被调用，执行两个阶段：
  - **Phase 1**: 更新结构化记忆库（Status / Knowledge / Procedural entries）
  - **Phase 2**: 输出 `<context_for_action>` 提醒或 `<no_intervention/>`

### 记忆库结构
```
记忆库包含三类条目:
- Status entries: 当前任务状态
- Knowledge entries: 环境事实、已验证信息
- Procedural entries: 失败尝试、诊断结果、未完成子目标
```

### 关键机制
- **选择性干预** > 被动记忆暴露 > 始终注入 > 仅顾问指导
- 记忆 Agent 是可插拔的，支持搭配任意 Action Agent（Claude Sonnet 4.5、Opus 4.6 等）
- 使用 Claude Opus 4.6 作为 Memory Agent 驱动大部分实验

## 实验结果

| 设置 | Terminal-Bench 2.0 | τ²-Bench |
|------|------|------|
| Sonnet 4.5 基线 | 37.6% | 55.0% |
| + Memory Agent | **45.9%** (+8.3 pp) | **61.8%** (+6.8 pp) |
| Opus 4.6 基线 | — | — |
| + Memory Agent | +2.4 pp | +2.5 pp |

### Ablation 结论
- 选择性干预优于被动记忆暴露、始终注入、仅顾问指导和通用检索
- Qwen3.5-27B (SFT+GRPO) 在 SETA 上训练的策略可部分迁移到 Terminal-Bench

## 局限性
- Memory Agent 目前依赖 Claude Opus 4.6（闭源），限制可复现性
- Qwen3.5-27B 微调策略的跨任务泛化性仍有限
