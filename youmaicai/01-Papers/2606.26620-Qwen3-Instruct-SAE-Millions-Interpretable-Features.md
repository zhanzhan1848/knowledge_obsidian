# 🥬 LLM 论文分析: Qwen3-Instruct SAE — Millions of Interpretable Features

## 基本信息
- **arXiv**: [2606.26620](https://arxiv.org/abs/2606.26620)
- **类别**: cs.LG
- **日期**: 2026-06-25

## 核心贡献
1. **Qwen3-Instruct SAE Suite**: 覆盖 Qwen3-1.7B, Qwen3-4B, Qwen3-8B 的完整 SAE 套件
2. **分层 SAEs**: 在三个关键位置训练: residual streams / MLP outputs / attention outputs
3. **百万级特征**: 大规模可解释特征字典
4. **实用案例**: Refusal Steering - 用 SAE 特征因果性地 steering 模型拒绝行为

## SAE 架构

### 训练位置
| 位置 | 描述 | 规模 |
|------|------|------|
| Residual Stream | 层间残差连接 | 全层 |
| MLP Outputs | FFN 输出 | 全层 |
| Attention Outputs | 注意力输出 | 全层 |

### 模型覆盖
- **Qwen3-1.7B**: layer-wise SAEs (3 sites)
- **Qwen3-4B**: layer-wise SAEs (3 sites)
- **Qwen3-8B**: SAEs on subset of residual stream layers

## 评估指标

### 1. Activation-level Metrics
- Reconstruction error
- Sparsity scores

### 2. Model-level Recovery Metrics
- 下游任务性能恢复
- Feature interpretability

## 关键发现

### Sparsity-Fidelity Trade-offs
- 不同层和不同组件有不同的最优稀疏度-保真度权衡
- Residual stream SAEs 通常更密集
- Attention outputs SAEs 通常更稀疏

### 分层特征分布
- 浅层（early layers）: 基础语言特征（语法、词法）
- 中层（mid layers）: 语义特征（概念、实体）
- 深层（late layers）: 任务特征（推理、规划）

## Refusal Steering 案例研究

### 方法
1. 识别与 refusal 相关的 SAE 特征
2. 对选定特征进行激活值干预（amplify/suppress）
3. 观察模型行为变化

### 结果
- 选定的 SAE 特征可以**因果性地** steering Qwen3 的拒绝行为
- 不需要 fine-tuning，只需要激活值干预

### 核心洞察
> Sparse Autoencoders 不仅可以解释模型，还可以**控制模型**

## 与传统方法对比

| 方法 | 可解释性 | 因果控制 | 参数量 |
|------|----------|----------|--------|
| Attention Visualization | 中 | 无 | 0 |
| Activation Patching | 中 | 部分 | 0 |
| SAE (本工作) | 高 | 强 | ~SAE size |
| Fine-tuning | 低 | 强 | Full model / LoRA |

## 局限性
- SAE 训练本身计算开销大
- 特征的可解释性标注仍依赖人工
- Qwen3-8B 只在部分层训练了 residual stream SAEs
