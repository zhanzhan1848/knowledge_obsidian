# NightVision: Black-Box Inference of LLM Architectural Properties

## 元信息

| 标题 | Black-Box Inference of LLM Architectural Properties with Restrictive API Access |
|------|------|
| 作者 | Shreyas Chaudhari 等 |

| arXiv | arXiv:2607.01313 |
| 领域 | cs.LG, cs.AI, cs.CL, cs.CR |

## 核心贡献

1. **NightVision**：即使在受限 API（仅返回单一 logit）下，仍可恢复 LLM 多个架构参数
2. **Common Set Prompting**：多 prompt 暴露相同 output tokens 的 log probabilities
3. **Spectral Analysis**：从 log probabilities 推断 hidden dimension
4. **TTFT 测量**：利用首 token 时间推断 depth 和参数数量

## 方法

### 攻击表面
- 大多数商业 LLM 提供商已限制 API：仅暴露单个 logit，无法 bias logits
- 之前方法依赖 top-k logits 和 logit bias，已不再可用

### NightVision 技术栈
1. **Common Set Prompting**：构造多个 prompts 使相同 token 集合的 log probs 被暴露
2. **Spectral Analysis**：分析 log prob 矩阵推断 hidden dimension
3. **TTFT Timing**：端到端时间测量 + hidden dimension → depth + parameter count

## 实验结果

| 参数 | 恢复精度 |
|------|---------|
| Hidden dimension | 平均 23% 相对误差（MoE 模型 9%） |
| Depth & 参数 count | >3B 模型 53% 以内误差 |

## 安全启示

当前 LLM API 并未充分限制，无法完全隐藏底层架构细节

## 局限性

- 需要足够的 token budget
- 对非常小模型误差较大

## 建议

- **是否推荐**：是（安全研究）
- **适用场景**：LLM 安全审计、API 安全评估
