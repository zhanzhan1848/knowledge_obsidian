## 核心贡献

1. **RLVR（Reinforcement Learning with Verifiable Rewards）**：直接在目标环境中应用 RLVR，解决 LLM next-token 预测目标与 API 执行之间的目标不匹配问题
2. **Atlassian 工作流基准**：5 个合成环境，模拟 Jira REST v3 和 Confluence v2 API
3. **无需人工标签**：奖励完全从 tool-call trace 计算，无 live API、无 learned judge、无人类标注

## 方法

### 问题
- LLM 训练目标是预测下一个 token，不是精确执行 API 操作
- 在企业 SaaS 工作流中表现为：字段丢失、工具幻觉、单次读取后提前停止

### RLVR 方案
- 构建 schema-fidelity 合成环境
- 奖励完全基于 tool-call trace 的可验证奖励
- Qwen3-1.7B 和 Qwen3.5-4B 在相同 GRPO 训练检查器上评估

## 实验结果

| 场景 | 基础奖励 | RLVR 后 |
|------|---------|---------|
| 4个非退化场景 | 0.35-0.92 | **0.95-1.00** |
| Confluence page creation | 0.35 | **1.00** |

## 局限性

1. **可扩展性限制**：手工设计可验证奖励无法扩展到更多端点
2. **饱和场景**：ticket-transition 场景的奖励形状饱和，prompted 4B 已达最大值

## 建议

- **是否推荐**：是
- **适用场景**：企业 API 工具使用、Niche SaaS 工作流自动化
