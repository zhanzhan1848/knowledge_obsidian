# ProvenanceGuard: Safeguarding LLM Agents from Misalignment

## 元信息

| 标题 | Safeguarding LLM Agents from Misalignment through Provenance Analysis |
|------|------|
| 作者 | Yining She, Yiliang Liang, Eunsuk Kang |
| 链接 | [原文](https://arxiv.org/abs/2607.01236) |
| arXiv | arXiv:2607.01306 |
| 领域 | cs.CL, cs.AI |

## 核心贡献

1. **ProvenanceGuard**：多阶段 pipeline，在工具执行前检测三种类型的 misalignment
2. **基于 provenance 的概念框架**：形式化 misalignment 检测为判断工具调用是否被 agent 上下文中可追踪证据支持
3. **显著降低误判率**：在 Agent-SafetyBench 和 WorkBench 上评估

## 方法

### Provenance 分析框架
misalignment = proposed tool call 是否被上下文中可追踪证据支持

### 三种 Misalignment 类型
检测 agent 动作是否与用户意图一致，在工具执行前阻断

## 实验结果

| 基准 | LLM-as-Judge 误判率 | ProvenanceGuard 误判率 |
|------|------------------|---------------------|
| Agent-SafetyBench | 42.9% | **1.8%** |
| WorkBench | 32.1% | **17.3%** |

**干预负担**：从 30.5% 降至 12.8%（任务成功轨迹）

## 局限性

- 仅测试 10 个 backbone LLMs
- 三种 misalignment 类型的覆盖性待验证

## 建议

- **是否推荐**：是
- **适用场景**：LLM Agent 安全防护、工具调用场景、企业自动化
