# Kara: Efficient Reasoning LLM Serving via Sliding-Window KV Cache Compression

## 核心贡献

1. **Kara**：推理 LLM 服务的滑动窗口 KV Cache 压缩方法
2. **KvLLM**：基于 vLLM 的推理框架，整合 Kara + PagedAttention
3. **解决现有 KV 压缩的两个关键局限**：
   - 阈值触发压缩策略吞吐量改善有限，可能完全删除某些 block 的 KV
   - 固定边界保留策略无法保留任意位置的重要灵活大小 chunk

## 方法

### Sliding-Window 压缩
- 仅在最近生成的上下文中进行解码时压缩
- 利用双向注意力评分和选择信息丰富的 KV 对

### Token2Chunk 模块
- 将选定的 KV 对扩展为 chunks
- 支持灵活保留任意位置的重要语义信息块

### KvLLM
- 适配 PagedAttention
- 降低 KV cache 内存使用
- 有效提高输出吞吐量

## 局限性

- 需要在更多推理模型上验证
- 长程依赖任务上可能存在信息损失风险

## 建议

- **是否推荐**：是
- **适用场景**：长 CoT 生成的推理 LLM 服务、延迟敏感场景
