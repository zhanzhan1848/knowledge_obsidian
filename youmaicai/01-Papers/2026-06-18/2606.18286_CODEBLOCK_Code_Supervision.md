## 核心贡献

1. **问题**：Code LLM 的监督微调通常对所有响应 token 应用统一交叉熵损失，隐式假设每个 token 提供同等有用的学习信号。直接将 token 级选择方法迁移到代码会破坏语法和语义连贯的程序单元，因为代码依赖于结构完整性和定义-使用关系。

2. **CodeBlock**：提出结构感知稀疏监督框架，选择结构完整的代码证据而非孤立 token。

## 核心方法

### Pipeline
1. 选择高质量 instruction-response pairs
2. 将代码响应划分为语法连贯的 coding items
3. 通过在核心逻辑 token 上聚合广义交叉熵来估计其效用
4. 用数据流可达性和桥接信号重新排序，优先处理传播或连接重要程序依赖的 blocks

### 训练设置
- 完整响应保留作为上下文
- 损失仅应用于选定的代码项和信息性自然语言 token

## 实验结果

六个代码生成 benchmarks：
- CodeBlock 实现比 full-token SFT 更强的平均 pass@1
- 与竞争性选择 baseline 相媲美
- 仅使用 **1.9%** 的监督响应 token
