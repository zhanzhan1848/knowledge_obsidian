## 核心贡献
1. 提出 **SoftSkill**：将自然语言 Markdown skill 转换为紧凑连续上下文对象的方法
2. 冻结主干网络，通过 next-token 预测调优 soft skill，推理时部署为潜在行为先验
3. 仅用 32 个 soft token prefix 即可大幅超越无 skill prompting 基线
4. 揭示了 Agent skill 的新范式：**latent behavioral control** 而非 additional Markdown

## 方法

### 问题
- Agent skills 通常部署为自然语言 Markdown 文件
- 缺点：每个任务实例中，冻结语言模型必须将长文本 artifact 间接翻译为生成时行为
- 计算开销大，且 skill 信息未直接编码到模型参数中

### SoftSkill 方案
1. 将自然语言 skill 初始化为紧凑连续上下文对象（soft tokens）
2. 冻结基模型，通过 next-token 预测训练 soft delta
3. 推理时将 soft tokens 作为 latent behavioral priors 部署

## 实验结果
- **单轮主要设置**（Qwen3.5-4B）：
  - SearchQA: +8.3 points vs no-skill prompting
  - LiveMath: +42.1 points vs no-skill prompting
  - DocVQA: +1.3 points vs no-skill prompting
  - 相比 SkillOpt: +5.2 points (SearchQA), +12.5 points (LiveMath)
  - 用几十个 virtual tokens 替代数百到数千个 Markdown skill tokens

- **Agentic 执行**（更难边界情况）：
  - 稀疏轨迹模仿提供有用信号
  - 尚未鲁棒压缩长 horizo 程序行为

## 局限性
- 长 horizo 程序行为压缩仍不够鲁棒
- Agentic 设置下稀疏轨迹 imitation 信号有限

## 建议
- **是否推荐使用**：是（对 Agent skill 表示和高效 adaptation 有重要创新）
- **适用场景**：Agent skill learning, behavioral adaptation, efficient fine-tuning
- **关键词**：soft skill, behavioral compression, agent adaptation, prompt engineering, latent control
