# 🥬 LLM 论文分析：Query Visibility Changes KV-Cache Compression Rankings

## 基本信息
- **作者**：Daming Luo, Christy Liang, Junyu Xuan
- **发表**：arXiv — 2026-07-11
- **链接**：[原文](https://arxiv.org/abs/2607.11942) | arXiv:2607.11942

## 核心贡献
1. **核心问题**：KV-cache 压缩方法通常在 query-aware 协议下评估（query 先追加到 context 再压缩），但实际部署需要 query-agnostic（压缩发生在看到问题之前）
2. **大规模审计**：6 种压缩方法 vs 3 个 trivial baselines，144,300 paired evaluations (RULER-8192) + 40,800 (LongBench)
3. **三项关键发现**：
   - **Query-aware vs agnostic 改变排名**：agnostic 协议下，5 种共享 common attention backend 的方法中，只有 KeyDiff 持续击败 trivial baseline（31/36 cells）；广泛部署的 SnapKV 输给 "keep start + recent window"（-0.066）
   - **Per-method drop 与 query 可见性一致**：Delta 从 SnapKV 的 +0.198 到 KeyDiff 的 +0.011，Delta 大小与其 scoring signal 对问题的可见性排序一致

## 核心洞察
> KV-cache compression 的经济价值在于复用：一次压缩，多个问题复用。因此压缩必须在看到问题之前发生（query-agnostic）。

## 建议
- **适用场景**：KV-cache 压缩方案评估与部署决策
- **推荐程度**：⭐⭐⭐⭐⭐ — 揭示了 benchmark 评估与实际部署的严重不一致
