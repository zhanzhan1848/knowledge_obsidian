## 核心贡献

1. **Skill Collision 问题**：企业 AI Agent 将用户查询路由到专业技能时，当两个技能描述重叠会导致错误路由，称为 skill collision
2. **自动化描述优化pipeline**：在 9 个技能、372 回归案例的生产级企业群聊 Agent 上，F1 达 79.2%，与人工调优的 79.4% 持平，同时将每技能工程时间从 120 分钟缩短到 3.8 分钟（32倍加速）
3. **单次LLM重写即足够**：系统消融实验表明，任何可用假阳性和假阴性案例的单次LLM重写即可捕获大部分改进，其他设计选择（迭代预算、反馈信号组合、混淆对双重编辑、训练集大小）对最终F1影响均 < 0.5%
4. **Train-validation F1 gap 作为诊断信号**：可用于识别需要架构级干预而非文本级干预的情况

## 模型架构

- **路由系统**：基于 LLM 的自然语言技能描述匹配
- **优化 pipeline**：自动化描述优化框架

## 实验结果

| 指标 | 人工调优 | 自动化 pipeline |
|------|------|------|
| F1 | 79.4% | 79.2% |
| 工程时间/技能 | 120 min | 3.8 min |
| 加速比 | - | 32x |

- 在 ToolBench（16k 工具）上的系统消融验证了单次重写的充分性

## 局限性

- 无法解决两个技能意图范围本身重叠的情况（需要架构级干预）
- Train-validation F1 gap 大时提示需要架构调整

## 建议
- 是否推荐使用：**是**，生产级实用价值高
- 适用场景：企业 AI Agent 技能路由、工具调用系统
