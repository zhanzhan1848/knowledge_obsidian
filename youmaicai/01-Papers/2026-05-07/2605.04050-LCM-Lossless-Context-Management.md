## 核心贡献

### 问题
现有 LLM 记忆方案（无论是压缩还是递归）都面临**上下文丢失**或**检索不可靠**的问题。

### LCM 方案
**两大确定性、引擎管理的机制**:

1. **Recursive Context Compression**:
   - 分层摘要 DAG 自动压缩旧消息
   - 保留到每个原始消息的**无损指针**
   - 不丢失任何历史状态

2. **Recursive Task Partitioning**:
   - LLM-Map 等引擎管理的并行原语替代模型自写循环
   - 类比: 从 GOTO → 结构化控制流
   - 获得终止保证、短任务零开销

### 与 RLM 的关系
- 可视为 Recursive Language Models (RLMs) 的扩展和验证
- 但将符号化递归分解为确定性引擎管理机制

## 实验结果

- 编码 agent **Volt** (LCM-augmented, Opus 4.6)
- **OOLONG long-context eval** (32K~1M tokens): 全面超越 Claude Code

## 核心洞察

> 从 GOTO 到结构化控制流的类比非常深刻——牺牲最大灵活性换取**终止保证**、**零开销连续性**和**无损可恢复性**
