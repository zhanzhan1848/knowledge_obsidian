## 核心贡献

1. **OpenTools 框架**: 社区驱动的工具箱，统一工具 schema 标准，提供轻量级即插即用包装器

2. **双精度问题识别**: 指出 Agent 工具调用失败源于两个方面：
   - **工具使用精度** (Tool-use Accuracy): Agent 调用工具的能力
   - **工具内在精度** (Intrinsic Tool Accuracy): 工具本身正确性
   - 现有工作主要关注前者，忽略了后者

3. **自动化测试套件**: 对工具进行自动化测试和持续监控

4. **公开 Web Demo**: 用户可运行预定义 Agent 和工具，并贡献测试用例

5. **性能提升**: 高质量任务特定工具相对现有工具箱提升 **6%-22%**

---

## 核心问题

**工具集成 LLM 的可靠性瓶颈**: 工具集成 LLM 可通过外部工具进行检索、计算和真实世界操作，但可靠性仍是关键瓶颈。

**现有研究的盲点**: 大多数工作强调工具使用精度（Agent 如何调用工具），而忽略了工具内在精度（工具本身是否正确）。

---

## 框架组成

1. **核心框架**: 标准化工具 schema
2. **初始工具集**: 提供高质量基础工具
3. **评估管道**: 自动化测试和持续监控
4. **贡献协议**: 社区驱动的工具扩展机制

---

## 关键发现

- 工具内在精度与工具使用精度同样重要
- 社区贡献的高质量任务特定工具可带来 **6%-22%** 的相对提升
- 提升在多种 Agent 架构和下游任务上具有普遍性

---

## 局限性

- 社区贡献质量控制挑战
- 工具 schema 标准化需要社区共识
