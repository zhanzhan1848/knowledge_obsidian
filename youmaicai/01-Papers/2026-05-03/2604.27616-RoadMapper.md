## 核心贡献
1. **RoadMap基准**：首个研究路线图生成任务基准，包含10个研究领域、5种研究类型、中英双语
2. **多智能体系统RoadMapper**：模拟人类专家流程，包含6个LLM驱动的智能体
3. **迭代"批判-修订-评估"机制**：通过DPO训练的评估智能体判断路线图质量并迭代优化
4. **效率提升**：节省84%的人工专家时间，性能提升8%+

## 模型架构

### RoadMap基准
- **Skill-Repo**：8,436个专业技能点（名称、描述、解决的问题）
- **Golden-Roadmap**：1,705个复杂研究问题及专家设计的路线图
- 数据来源：ProQuest和CNKI的学位论文

### RoadMapper架构
```
Init Agent → Knowledge Agent → Critique-Revise-Evaluate循环
     ↓              ↓                   ↓
  生成初稿      知识增强          多轮迭代优化
```

### 6个智能体
1. **Init Agent (I)**：接收研究问题，生成初始路线图
2. **Knowledge Agent (K)**：从Skill-Repo检索相关技能点，增强路线图
3. **Logic Critique Agent (L)**：分析节点间的逻辑关系，输出修订建议
4. **Granularity Critique Agent (G)**：分析任务分解粒度（过细/过粗）
5. **Revise Agent (R)**：根据批判建议修订路线图
6. **Evaluate Agent (E)**：评估路线图质量（DPO训练）

### 迭代流程
```
y_initial = I(x_problem)
y_0 = K(knowledge, y_initial)
repeat:
    LC_t = L(y_t)      # 逻辑批判
    GC_t = G(y_t)      # 粒度批判
    y_{t+1} = R(y_t, LC_t, GC_t)
    E_t = E(y_{t+1})
until E_t >= passing_score or max_iterations
```

## DPO训练

### 偏好数据构建
- 使用Qwen3-32B为每个路线图生成10个候选评估
- 7位专家投票选择最优(y_w)和次优(y_l)作为偏好对
- 最终偏好数据集：818个路线图的偏好对

### DPO目标函数
```math
L_{DPO} = -E_D[log σ(β log π_θ(y_w|x)/π_ref(y_w|x) / (π_θ(y_l|x)/π_ref(y_l|x)))]
```

## 评估指标

| 指标 | 说明 |
|------|------|
| DegreeScore | 出度差异 (1 - |Deg_g - Deg_o|/Deg_g) × 100 |
| DepthScore | 深度差异 (1 - |Dep_g - Dep_o|/Dep_g) × 100 |
| StepScore | 关键步骤得分（GPT-4o mini评估） |
| LogicScore | 内部逻辑一致性得分 |

## 实验结果

### 主要性能对比
| 模型 | 方法 | 英文Avg | 中文Avg |
|------|------|--------|--------|
| Qwen3-235B-A22B | RoadMapper | **79.14** | **80.08** |
| DeepSeek-V3.2 | RoadMapper | **79.14** | **79.61** |
| Llama 3.3 70B | RoadMapper | 70.69 | 70.20 |

### 消融实验
- 解决Q1（专业知识不足）：StepScore提升8.24
- 解决Q2（任务分解不合理）：结构指标提升7.04
- 解决Q3（逻辑关系混乱）：LogicScore提升7.79

### 实现细节
- 最大迭代次数：5
- 通过分数：80
- 最大检索技能点数：30
- 评估智能体：Qwen3 32B (DPO训练)

## 局限性
- 依赖Skill-Repo的质量和覆盖范围
- 多智能体协作可能带来额外的计算开销

## 建议
- **是否推荐使用**：是，尤其适合科研辅助、复杂问题规划
- **适用场景**：科研路线图设计、教育辅导、复杂任务分解
- **改进方向**：扩展研究领域覆盖、提升多模态支持
