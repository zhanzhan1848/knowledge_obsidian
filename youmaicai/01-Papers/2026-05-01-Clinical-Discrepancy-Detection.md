# Detecting Clinical Discrepancies in Health Coaching Agents: A Dual-Stream Memory and Reconciliation Architecture

## 基本信息
| 标题 | 值 |
|------|-----|
| 作者 | Samuel L Pugh, Eric Yang, et al. |
| 链接 | [原文](https://arxiv.org/abs/2604.27045) |
| arXiv | arXiv:2604.27045 |

## 核心贡献
1. 研究 LLM Agent 从单会话工具向管理纵向医疗旅程的持久系统转变中的记忆架构挑战
2. 提出**双流记忆架构**，严格分离患者叙述和结构化临床记录（FHIR）
3. 由专用**协调引擎**治理，评估每个提取记忆与患者 FHIR 档案的差异

## 问题背景

### 两个不完美的真实来源
1. **患者自我报告**: 当前但容易出现回忆偏差
2. **电子健康记录 (EHR)**: 医学验证但经常过时

### 通用 Agent 记忆系统的问题
- 通过用用户最新陈述覆盖旧事实来优化一致性
- 应用于临床数据时可能导致安全失败

## 方法

### 双流记忆架构
```
患者叙述流 ←→ 协调引擎 ←→ FHIR 结构化记录
```

### 协调引擎
- 评估每个提取记忆与患者 FHIR 档案的差异
- 按类型、严重程度和涉及的具体 FHIR 资源对差异进行分类

## 实验

### 数据集
- 26 个患者
- 675 个纵向健康 coaching 会话
- 混合数据集：真实提供者-患者记录 + 合成 FHIR 临床场景

### 结果
- **84.4%** 的设计的临床差异被检测到
- **86.7%** 的安全关键召回率
- **13.6%** 误差级联被直接量化
  - 追溯到对话中从非结构化对话提取记忆时丢失的临床细节

## 结论
验证患者报告的记忆与临床记录的对比对于安全部署纵向健康 Agent **既可行又必要**。
