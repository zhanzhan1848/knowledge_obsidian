## 核心贡献

1. **LoRA微调超越专用模型**: 在小规模高质量数据集（MBPP）上LoRA微调的Code Llama 7B超过更广泛微调的Code Llama-Python-7B（40.1% vs 38.4% pass@1）
2. **Sophia vs Adam**: Sophia优化器收敛更快，但最终pass@1分数差异不大
3. **傅里叶正则化显著提升跨语言迁移**: Fourier-based正则化将Java任务pass@1从34.2%提升到42.1%

## 问题背景

企业环境中通常需要多种编程语言，但为每种语言单独微调大语言模型计算成本高昂。

研究问题：参数高效微调方法（LoRA）+ 优化器增强能否改善从Python到Java等语言的跨语言代码生成？

## 方法

### 基础设置
- 基础模型: Code Llama 7B
- 微调方法: LoRA (Low-rank Adaptation)
- 优化器对比: Adam vs Sophia
- 训练数据集: MBPP (Mostly Basic Python Problems)

### Fourier-based Regularization

在微调过程中引入频域正则化，利用傅里叶变换的特性来约束模型参数。

### 实验设计

1. 跨语言任务: Python → Java
2. 评估指标: pass@1
3. Baseline对比: Code Llama-Python-7B

## 实验结果

| 方法 | Python pass@1 | Java pass@1 |
|------|---------------|--------------|
| Code Llama-Python-7B baseline | 38.4% | 34.2% |
| LoRA + MBPP | **40.1%** | 35.8% |
| LoRA + MBPP + Sophia | 40.5% | 36.1% |
| **LoRA + MBPP + Fourier Regularization** | 40.8% | **42.1%** |

## 核心发现

1. **小规模高质量数据可超越大规模专用模型**: MBPP上LoRA微调的Code Llama 7B超过Code Llama-Python-7B
2. **Sophia优化器收敛更快但最终性能相近**: 收敛速度优势明显，但最终pass@1差异不大
3. **Fourier正则化显著改善跨语言迁移**: Java任务提升7.9个百分点（34.2%→42.1%）

## 结论

结合LoRA、优化训练方法和频域正则化，可以高效地将单一语言LLM适配到多种编程语言。
