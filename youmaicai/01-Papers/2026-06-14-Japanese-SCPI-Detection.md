# Japanese SCPI Detection: 日语预训练语料中的敏感个人信息检测

## 核心贡献

1. **首次研究**: 首个探索日语特殊关怀需求个人信息 (SCPI) 检测的研究
2. **数据集**: 使用 LLM 构建 SCPI 日语数据集
3. **分类器**: 训练机器学习模型实现高效的 SCPI 检测
4. **发现**: 揭示了日语 SCPI 准确检测的挑战

## 背景

### 隐私保护法规背景
- **日本《个人信息保护法》(APPI)**: 定义了特殊关怀需求个人信息 (SCPI)
- SCPI 包括：残障信息、医疗记录、犯罪前科等需要特殊处理的个人信息

### 研究动机
- 预训练语料可能包含敏感个人信息
- 检测和过滤对合规和防止信息泄露至关重要
- 英语等语言的敏感信息检测已有研究，日语研究不足

## 方法

### LLM 辅助标注流程
```
日语语料 → LLM 标注 → SCPI 标注数据集 → ML 模型训练 → SCPI 分类器
```

### 模型选择
- **LLM**: 用于高质量标注（相比传统规则方法更灵活）
- **轻量级 ML 模型**: 用于大规模高效推理
  - 传统 ML 模型（高效）
  - BERT-based Transformer（高精度）

### BERT 在日语 NLP 中的应用
```
Vaswani et al. (2017) Transformer → BERT 架构 → 日语文本处理
```

## 局限性

1. **日语特殊性**: 日语书写系统复杂（汉字、平假名、片假名混合）
2. **SCPI 定义模糊性**: 某些 SCPI 边界不清晰
3. **大规模推理成本**: 全语料库 LLM 推理代价高昂
4. **检测准确性挑战**: 日语语境下的 SCPI 检测难度大

## 建议
- **适用场景**: 日语 LLM 预训练语料隐私合规
- **方法论价值**: LLM 标注 + 轻量模型的两阶段方案可迁移到其他语言
- **后续方向**: 提升检测准确率，探索更多日语特性
