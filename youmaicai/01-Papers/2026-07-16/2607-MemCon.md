# 🥬 LLM 论文分析：MemCon

## 基本信息
- **标题**：Memory as a Controlled Process: Learned Adaptive Memory Management for LLM Agents
- **作者**：Eric Hanchen Jiang et al.
- **发表**：arXiv:2607.13591 | cs.CL / cs.AI
- **链接**：[原文](https://arxiv.org/abs/2607.13591)

## 核心贡献

1. **将记忆操作建模为 MDP**：将 Agent 的记忆读写决策问题形式化为马尔可夫决策过程

2. **自适应策略**：学习何时检索、检索什么/多少、何时注入蒸馏计划、何时整合/遗忘

3. **无需预训练、无额外 LLM 调用**：使用轻量级 contextual bandit + UCB 探索

4. **后端无关**：可包装任意现有记忆实现

## 关键洞察

### 静态记忆系统的根本瓶颈
- 任务早期：记忆稀疏，最小化检索最优
- 目标重复出现：复用计划 > 最近邻检索
- Agent 卡住：换查询重新检索
- 长任务流：需要整合和剪枝记忆存储

→ 最优记忆行为是**上下文相关的**，固定启发式无法适应

## 实验结果

| 指标 | MemCon vs 基线 |
|------|---------------|
| 任务成功率 | 提升 **15.2 points** |
| Token 消耗 | 降低 **5-20%** |

覆盖 6 个基准、3 种 Agent 框架、3 种 LLM 后端。

## 局限性
- 依赖 task-by-task binary feedback，可能在某些场景难以获得
- UCB 探索在极窄最优解附近可能效率低

## 建议
- **推荐使用**：是
- **适用场景**：长期任务 Agent、需要多轮交互的系统
- **亮点**：首次将记忆管理从「手工启发式」推进到「可学习策略」
